# ハードウェア構成まとめ

## 今回の主構成

今回のArduino実機コードは、白線/黒床コースを走る「前輪サーボ操舵 + 後輪DCモーター駆動」を主構成にしています。

| 要素 | 役割 | コード上の扱い |
|---|---|---|
| Arduino UNO | 制御本体 | `arduino:avr:uno` |
| S1-S4カラーセンサー | RGBのG出力で白線位置を検出 | A0-A3へ直接入力 |
| 前輪サーボ | 進行方向を操作 | `Servo.write()` |
| 後輪DCモーター | 前進、停止、短時間後退 | PWM + DIR |
| 超音波センサー | 障害物距離 | TRIG/ECHOデジタルピン |
| MPU-6050 | roll角検出 | I2C A4/A5 |

カラーセンサー4つでA0-A3、MPU-6050でA4/A5を使うため、Arduino UNOのアナログ系ピンA0-A5をすべて使う構成です。カラーセンサーをMUXで切り替える構成は使いません。

## 旧メカナム構成との差分

以前の説明には4輪メカナムと4個DCモーターの前提がありました。今回の主スケッチではメカナム横移動は使わず、障害物時は一度停止し、サーボを切って低速前進または停止維持する安全寄りの動作にしています。

`cpp/` は制御ロジック学習用の旧サンプルとして残しています。実機で書き込む対象は `arduino/mechatro_robot_control/mechatro_robot_control.ino` です。

## 電源注意

- サーボとDCモーターはArduino 5Vピンから直接大電流を取らない。
- Arduino、サーボ電源、モーター電源、センサー電源のGNDを共通化する。
- 最初の試験ではタイヤを浮かせる。
- サーボ可動範囲は機械的にぶつからない角度へ調整する。
